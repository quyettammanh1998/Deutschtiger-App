## Deferred (shared-token/l10n/out-of-scope)
- **`lib/core/design_tokens.dart` not edited** (per constraint) — `AppColors.primary` is `#FF8FA3` (rose), not web's `hsl(32,93%,54%)` orange `--primary`. User bubble color therefore doesn't hex-match web's orange primary; it matches the app's own already-established primary token everywhere else in the Flutter app. This is a pre-existing app-wide token difference, not new drift from AI chat.
- **`lib/l10n/*.arb` not edited** (per constraint) — no dedicated AI-chat placeholder/label keys exist. Reused `socialTypeMessageHint` for the composer placeholder; all other new-facing strings (header title, chip labels, welcome text, button labels) are screen-local Dart literals matching web's copy verbatim, same approach as the pre-existing file.
- **`lib/widgets/ai/chat_history_sidebar.dart` not restyled** — imported/used as-is (out of file-ownership scope: not "directly-embedded" in `ai_chat_page.dart`, lives in its own file). Still uses `Colors.white`/`Colors.grey[...]` instead of `AppColors` tokens — visible drift if sidebar is opened. Flag for a follow-up pass with explicit scope.
- **Feature-action chips** (`FEATURE_ACTIONS` — post-reply deep-link buttons to Flashcard/Games/Exams/etc., web `FeatureActions` component) — not implemented. Requires mapping ~14 web routes to Flutter `GoRouter` paths; out of budget for this pass and not explicitly named in the composer/header/bubble/chip scope.
- **Image attach button** — web has a paperclip button (multi-image upload via Supabase Storage). Flutter's `AiChatNotifier.sendMessage` accepts `attachments` but no upload flow/UI exists yet in the app; not added here to avoid introducing unwired/fake behavior. User-bubble attachment *rendering* was added (for when attachments do exist, e.g. future upload support or session history with images).
- **Popup/drawer mode, voice input, session-limit/session-full inline cards, markdown rendering** (`react-markdown`) — web's `AiChatPanel` also renders full Markdown for assistant replies; Flutter renders plain `Text`. Out of scope (would need a markdown package decision) — flagged for a follow-up if content fidelity (not just chrome) is required.
