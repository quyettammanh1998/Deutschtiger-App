# Flutter live-data inventory

Baseline: 2026-07-15. This inventory follows every route family mounted by
`lib/navigation/app_router.dart`. It distinguishes an actual production source
from a screen that merely renders successfully. “Blocked” means the route must
not be presented as release-ready until the listed owner resolves it.

## Classification

- **Live** — uses an authenticated/public API contract already documented in
  `flutter-api-contract-matrix.md`, or a versioned local legal document with no
  user data.
- **Fixture-only** — data is bundled for preview/demo or a route supplies a
  fallback object; it must not make a production learning-data claim.
- **Feature-flagged** — route may remain in the source tree but must be hidden
  from release navigation until it has a verified source.
- **Blocked** — an enabled route currently uses mock/placeholder data or a
  contract gap. This is a release blocker, not a feature flag implementation.

## Offline policy: current implementation

The app is **online-first**. `OfflineService` reports connectivity only; it does
not write a cache, persist review data, or attempt a fictional sync. On a lost
connection the shell displays a recoverable offline banner and the feature
surfaces their own retry/error state.

No durable offline write may be added until the backend contract defines all of:

1. a client-generated `operation_id`;
2. server-side idempotency and ownership enforcement;
3. conflict/version response semantics; and
4. logout purge, sensitivity classification, TTL and stale-state UI if a
   read-only cache is approved.

## Release gates created by this inventory

- Release navigation and deep links must not expose a route marked **Blocked**
  unless its matching compile-time QA gate is explicitly enabled.
- `test/structure/release_live_data_guard_test.dart` rejects direct
  `mock`/`fixture`/`placeholder` markers in the mounted release-visible route
  screens. It is intentionally scoped to route-entry screens rather than a
  global grep so fixtures remain possible in tests, previews and disabled route
  families.
- A route marked **Feature-flagged** needs a central visibility gate before it
  can be shown to users.
- Every replacement of fixture/mock data must add its endpoint to
  `docs/flutter-api-contract-matrix.md` plus client and handler tests.
- Settings cannot offer a completed export until an authenticated account-export
  endpoint, native save/share handling and authorization tests exist.
